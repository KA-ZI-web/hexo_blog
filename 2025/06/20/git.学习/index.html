记录一些我的git远程仓库的学习

这是一篇从0开始的小白学习git的学习笔记（记录我学习过程中遇到的错误以及解决方法）

正文开始

下载git

  • 找到git的下载链接 ： https://git-scm.com/downloads ，点击进去；
  • 选择对应的系统和版本进行安装
  • 这里我选择的是windows
  • 等待下载ing…
  • 安装这里不做过多介绍了

** 建立密钥**

在进行Git版本控制时，HTTP和SSH是两种常见的传输协议，这里采用保密性较强的ssh协议进行操作

  • 1、打开github->点击头像->选择settings->选择SSH and GPG keys->点击NEW SSH Key->
  • 2、在本地建立密钥：
    • 打开刚才下载的git bash
    • 打开本地的.ssh目录:
    • 生成本地公钥：
    • 再次进入.ssh目录，并打开目录会发现生成了一个id_rsa.pub文件，这个pub文件里面就保存着本地的公钥信息
  • 3、将公钥信息复制粘贴到第一步操作中在github中创建的密钥中
  • 点击Add SSH key 后添加成功！

** 配置本地项目 **

  • 这里介绍一个Git概念： 工作区、暂存区、版本库和远程仓库
    • 工作区：就是你电脑里面可以看见的目录
    • 暂存区：英文叫 stage 或 index。一般存放在 .git 目录下的 index 文件（.git/index）中，所以我们把暂存区有时也叫作索引（index）。
    • 版本库/本地仓库：工作区有一个隐藏目录 .git，这个不算工作区，而是 Git 的版本库。
    • 远程仓库（remote）: 托管在远程服务器上的仓库
  • 在正式开始操作之前，我们要执行一个操作：配置用户信息（用户名和邮箱），用于标识代码提交者。这一步操作是为了确保后续的提交操作正确进行。这里是用了全局操作，要是只对当前仓库进行配置可以去掉–global。
    • 用git bash打开我们的项目根目录，假如我们的项目根目录为learn-git
    • 初始化git库，之后会在项目跟目中产生一个.git文件（若看不见确认是否显示隐藏项目），这个就是我们的建立的本地仓库啦
    • 用git对当前目录文件进行跟踪（将工作区文件加入到暂存区）
    • 提交文件（将暂存区文件加入到本地仓库中）
    • 查看当前文件状态
